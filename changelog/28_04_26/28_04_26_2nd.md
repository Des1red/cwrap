# Intent Cleanup Changelog
`2026-04-28` `v2.6.6`

## CHANGES

- Removed unused state fields from intent handlers:
  - `fetchState.content`
  - `fetchState.debug`
  - `sendState.debug`
  - `uploadState.debug`
  - `uploadState.hasFile`

- Cleaned `fetch` redirect behavior:
  - Removed `follow=true` / `follow=false` semantic handling.
  - Removed default `--follow` emission from `FetchHandler.Translate`.
  - Added support for `nofollow` / `no-follow` semantic words mapping to `--no-follow`.
  - Fetch now follows redirects by default through runtime behavior, not by emitting `--follow`.

- Simplified `fetch` key/value handling:
  - Removed `profile=...` and `as=...` as semantic profile shortcuts.
  - `key=value` in `fetch` now consistently means query parameter, except for explicit supported shortcuts like `proxy=...`.

- Updated global semantic words:
  - Removed `follow` and `nofollow` from `isBooleanWord`.
  - Kept global semantic words limited to shared flags like `csrf` and `auto-cookie`.

- Cleaned `upload` debug handling:
  - Removed `--debug` emission from `UploadHandler.Translate`.
  - Kept debug behavior in `UploadHandler.ApplyDefaults` via `f.Debug = true`.

- Updated intent tests to match the new behavior:
  - `fetch firefox` now expects `--as firefox` without `--follow`.
  - `fetch nofollow` now expects `--no-follow`.
  - `fetch follow=false` now becomes a query parameter.
  - `fetch profile=chrome` now becomes a query parameter.
  - `upload` translation no longer expects `--debug`.
  - `recon` tests no longer use follow-related flags.

## REASON

The intent package was becoming hard to reason about because multiple input styles were doing the same thing. For example, profile selection could happen through:

```bash
firefox
browser
as=firefox
profile=firefox
--as firefox
```

Redirect behavior also had multiple competing forms:

```bash
follow
nofollow
follow=true
follow=false
--follow
--follow=false
--no-follow
```

This made the code harder to read and increased the number of paths that needed to be tested and maintained.

The cleanup moves toward a clearer rule:

```txt
bare words  = semantic shortcuts
flags       = explicit configuration
key=value   = request data or query/form data
```

## PROBLEM

The old behavior mixed command configuration with request data.

For `fetch`, this was confusing because most `key=value` values became query parameters, but some special keys changed command behavior:

```bash
page=2          # query parameter
profile=chrome  # command configuration
follow=false    # command configuration
```

That means `key=value` did not have one consistent meaning.

There was also duplicate/default behavior split between `Translate` and `ApplyDefaults`. For example, upload emitted `--debug` during translation while also setting `f.Debug = true` during defaults.

Redirect behavior was also mismatched with the current model. The model uses:

```go
NoFollow bool
```

but older intent code still emitted follow-style flags such as `--follow` or `--follow=false`.

## FIX

The intent behavior was simplified around clearer ownership:

- `Translate` converts semantic user input into explicit flags only when needed.
- `ApplyDefaults` owns command defaults such as method and debug preview mode.
- `key=value` is no longer used for general command configuration in `fetch`.
- Redirect disabling uses `--no-follow` / `nofollow` instead of `follow=false`.
- Shared semantic words no longer include fetch-specific redirect behavior.

The resulting behavior is easier to predict:

```bash
cwrap fetch http://x firefox
```

means:

```txt
GET request
Firefox request profile
follow redirects by default
```

```bash
cwrap fetch http://x firefox nofollow
```

means:

```txt
GET request
Firefox request profile
do not follow redirects
```

```bash
cwrap fetch http://x profile=chrome
```

now means:

```txt
query parameter: profile=chrome
```

For explicit configuration, use flags:

```bash
cwrap fetch http://x --as chrome
cwrap fetch http://x --no-follow
```

This keeps the intent layer smaller, the command language clearer, and the tests easier to understand.
