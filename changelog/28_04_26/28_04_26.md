# cwrap — Changelog
`2026-04-28` `v2.6.5`

---

## [REFACTOR] `builder` package renamed to `curl`

**Files:** `internal/builder/` → `internal/curl/`, all import paths updated

### Reason

The package was named `builder` generically but its sole purpose is constructing and executing `curl` commands. Renaming to `curl` makes the dependency graph self-documenting. `runner` and any other consumers updated their import paths accordingly.

## [FEAT] Cookie session integration — `send` auto-syncs to recon session store

**Files:** `internal/curl/sync.go`, `runner/runner.go`

### Problem

`send` wrote cookies to `~/.cwrap_cookies.txt` (home dir). Recon reads sessions from `~/.config/cwrap/sessions/<host>.json`. The two stores were completely separate — logging in via `send --auto-cookie` had no effect on subsequent recon runs, requiring manual `cookie:name=value` flags every time.

### Fix

**1. Jar path moved to config dir:**

```go
func jarPath() string {
    dir, _ := os.UserConfigDir()
    base := filepath.Join(dir, "cwrap")
    os.MkdirAll(base, 0755)
    return filepath.Join(base, "cookies.txt")
}
```

**2. `SyncJarToSession` reads the Netscape jar and writes all cookies into the session store after each `auto-cookie` request.**

The critical fix was handling the `#HttpOnly_` prefix — curl's Netscape jar format prefixes HttpOnly cookie lines with `#HttpOnly_<domain>`, causing them to be silently skipped as comments by a naive parser. `auth_token` and `refresh_token` are both HttpOnly and were never synced without this fix.

```go
if strings.HasPrefix(line, "#HttpOnly_") {
    line = line[len("#HttpOnly_"):]
    httpOnly = true
} else if strings.HasPrefix(line, "#") || strings.TrimSpace(line) == "" {
    continue
}
```

**3. `runner.go` calls `SyncJarToSession` after successful execution:**

```go
if req.Flags.AutoCookie {
    if err := curl.SyncJarToSession(req.URL); err != nil {
        fmt.Fprintf(os.Stderr, "\ncwrap: warning: failed to sync cookies: %v\n", err)
    }
}
```

### Result

```
cwrap send http://localhost:8081/api/login email=1@1.com password=1 json auto-cookie --run
# → auth_token, refresh_token, csrf_token written to ~/.config/cwrap/sessions/localhost.json

cwrap recon http://localhost:8081
# → session loaded automatically, role=member uid=1 across all entities
```

All three cookies — including the HttpOnly JWTs — land in the session store with correct `http_only`, `secure`, and `path` metadata.

## [FEAT] Multi-cookie parsing — comma-separated values in single token

**File:** `intent/scanner.go` — `Scan()`

### Problem

`cookie:name=value` only supported one cookie per token. Passing multiple cookies required repeating `cookie:` for each.

### Fix

The `TokenCookie` branch now splits on `,` and emits one token per pair:

```go
if strings.HasPrefix(lower, "cookie:") || strings.HasPrefix(lower, "cookies:") {
    kv := a[strings.Index(a, ":")+1:]
    for _, pair := range strings.Split(kv, ",") {
        pair = strings.TrimSpace(pair)
        if j := strings.Index(pair, "="); j != -1 {
            out = append(out, Token{
                Type:  TokenCookie,
                Key:   pair[:j],
                Value: pair[j+1:],
                Raw:   pair,
            })
        }
    }
    continue
}
```

Multiple separate `cookie:` tokens continue to work — both forms are equivalent.

### Usage

```
cwrap recon http://localhost:9001 cookie:auth_token=xyz,csrf_token=abc,refresh_token=def
```
