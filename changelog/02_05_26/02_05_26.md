# Changelog

```02-05-26``` ```v2.7.3```

## 1. Static assets not captured

**Problem:** Static asset URLs were silently dropped during expansion.

**Reason:** `isStaticAsset` returned early without registering the URL anywhere.

**Fix:** Added `k.StaticAssets map[string]bool` to `Knowledge`, `AddStaticAsset` method with query stripping, called at both `Expand` and `handleJSEndpoints` callsites.

---

## 2. JS files never fetched

**Problem:** `<script src="...">` tags were never queued for fetching.

**Reason:** Inverted condition `if !jsintel.IsPhantomJSURL` broke on every real JS file.

**Fix:** Removed the `!` — `if jsintel.IsPhantomJSURL` now correctly skips only phantoms.

---

## 3. Subdomains not discovered

**Problem:** `ftp.wingdata.htb` was silently dropped despite being in the HTML.

**Reason:** `normalizeLink` rejected any host that wasn't an exact match to base host.

**Fix:** Added `linkutil.SameSite` shared package using root domain comparison — both engines updated.

---

## 4. Minified JS variables matching `.test` TLD

**Problem:** `i.test`, `K.test`, `v.test` reported as internal domains.

**Reason:** `reInternalDomain` matched any suffix before `.test` regardless of length or case.

**Fix:** Raised minimum label length to `{3,}` in regex, added lowercase-only filter in the match loop — camelCase/PascalCase identifiers rejected.

---

## 5. POST requests sent with no body

**Problem:** Form endpoints were never probed with real POST data.

**Reason:** `transport.Build` always passed `nil` body, `Probe` had no body fields.

**Fix:** Added `Body []byte` and `ContentType string` to `Probe`, wired `req.Flags.Body` in `runQueuedProbes`, added `requeueForms` pass after main probe loop, cleared `req.Flags.Body = ""` before each probe to prevent base request contamination.

---

## 6. Server tech headers not captured

**Problem:** `Server`, `X-Powered-By` etc. were never recorded.

**Reason:** `httpintel.Learn` only tracked auth and content-type headers.

**Fix:** Added `Tech map[string]string` to `HTTPIntel`, populated in `httpintel.Learn` for a fixed list of fingerprinting headers, surfaced in global stats (deduplicated) and per-entity in report.

---

## 7. ProbeCount not incremented on transport failure

**Problem:** `ftp.wingdata.htb` showed `Probes: 0` despite being probed 12 times.

**Reason:** `ProbeCount++` was inside the identity loop gated on `executed` flag, which never fired if all transports failed.

**Fix:** Moved `ProbeCount++` before the identity loop, removed `executed` tracking entirely.

---

## 8. Spurious `IDLikeParam` on transport-dead entities

**Problem:** `ftp.wingdata.htb` carried `IDLikeParam` signal despite no HTTP response ever being received.

**Reason:** Param classification ran before the identity loop, tagging entities unconditionally regardless of transport outcome.

**Fix:** Split query appending (before loop) from param classification (after loop), gated classification on `len(identityStatuses) > 0`.

---

## 9. Non-HTTP services silently invisible

**Problem:** `ftp.wingdata.htb` appeared in the report with no context on why it had no data.

**Reason:** No finding or next step was derived for transport-failed entities.

**Fix:** Added condition in `deriveFindings` and `deriveNextSteps` — when `ProbeCount > 0` and no methods or statuses recorded, surfaces "non-HTTP service" finding and manual probe next steps.
