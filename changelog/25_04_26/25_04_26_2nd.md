# cwrap — Changelog
`2026-04-25` `v2.6.0`

## Overview
JWT tampering probe — stage 1 vulnerability confirmation and stage 2 chain
expansion for JWT-based authentication bypass. Three techniques, role
enumeration from report, tampered token vault for cross-probe sharing.

---

## New: JWT Tampering Probe (`probe/probe_jwt.go`)

Queued in `buildQueue` when any entity has `RoleBoundary` signal and the
report contains JWT tokens (vault or session cookies).

Priority 110 — runs before all other probes so confirmed tampered tokens
are available to IDOR, Ownership, and Credentialless probes.

### Techniques

| Technique | What it tests |
|-----------|--------------|
| `alg:none` | Server accepts unsigned tokens |
| `role-swap:<role>` | Server skips signature re-validation after claim change |
| `alg:none+role-swap:<role>` | Combined — unsigned token with elevated role injected |

### Role enumeration

Roles to attempt are collected in priority order:

1. Vault identity names — `"owner-uid-3"` → role `"owner"`
2. Entity identity records with role tags
3. Fallback hardcoded list: `admin, owner, root, superuser, staff, moderator`

Current role excluded from candidates. Only non-elevated current roles
trigger role swap attempts.

### Token source fallback

When `IdentityVault` is empty (single-user report with no discovered
live identities), the probe falls back to session cookies on entities:

```go
if len(p.Report.IdentityVault) == 0 {
    for _, ent := range p.Report.Entities {
        for _, sc := range ent.SessionCookies {
            // try all three techniques against session cookies
        }
    }
}
```

### Output — confirmed only

Non-confirmed attempts are silently recorded. Only confirmed bypasses
print to output:

```
testing: session [alg:none+role:admin] (role=admin uid=1)
token:   auth_token=eyJhbGciOiJub25lI...
[CONFIRMED] http://localhost:9006/api/admin → 200 (alg:none+role-swap:admin accepted)
response:   {"panel":"admin","status":"ok"}
```

### Architecture

```
Run()
├── collect RoleBoundary targets
├── for each identity in vault:
│   ├── tryAlgNone()
│   ├── tryRoleSwap()         — iterates CollectElevatedRoles()
│   └── tryAlgNoneWithRoleSwap() — iterates CollectElevatedRoles()
└── fallback: session cookies if vault empty

tryAlgNone / tryRoleSwap / tryAlgNoneWithRoleSwap
└── testJar()   — shared primitive: probe, record step, print on confirm
```

## New: JWT Chain Expansion (`chain/jwt.go`)

Seeds from stage 1 confirmed JWT steps. Re-applies the confirmed
technique and tests against role/admin boundary entities not yet covered.

```go
func expandJWT(session *result.Session, flags expandFlags)
func stage2JWTTargets(session *result.Session, seed result.Step) []report.ReportEntity
func applyTechnique(technique, token string) (string, error)
```

`applyTechnique` dispatches by technique string:

```go
case technique == "alg:none"                  → tokens.AlgNone
case strings.HasPrefix("role-swap:")          → tokens.RoleSwapTo
case strings.HasPrefix("alg:none+role-swap:") → tokens.AlgNoneWithRoleSwapTo
```

## New: Tampered Token Vault (`result.Session`)

Confirmed tampered tokens stored in `TamperedVault` so other probes
can use elevated tokens discovered by JWT tampering:

```go
type Session struct {
    Recon         *report.Report
    Steps         []Step
    TamperedVault map[string]map[string]string // identity/technique -> cookieName -> token
}

func (s *Session) StoreTampered(identity string, jar map[string]string)
func (s *Session) FindCookies(identityName string) map[string]string
func (s *Session) IdentityCookies(identityName string) map[string]string
```

`FindCookies` priority:
1. TamperedVault — elevated tokens take priority
2. IdentityVault in report
3. nil

All chain expanders (`expandIDOR`, `expandOwnership`, `expandCredentialless`)
and probes (`IDORProbe`, `OwnershipProbe`) updated to use
`session.FindCookies` instead of `tokens.FindIdentityCookies`.

## Validated on `localhost:9006` (`vulnAlgNone` mode)

```
alg:none               → 403  (token accepted, role still member)
role-swap:admin        → 401  (signature rejected)
alg:none+role-swap:admin → 200 CONFIRMED
  /api/admin       → {"panel":"admin","status":"ok"}
  /api/admin/users → [{"id":1,...},{"id":2,...}]
```

Test server (`testservers/jwt/jwt_server.go`) supports three modes:
- `vulnAlgNone`  — accepts unsigned tokens
- `vulnRoleSwap` — accepts role-swapped tokens with any non-empty signature
- `vulnNone`     — secure control
