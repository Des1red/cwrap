## Two-Stage Architecture

### Stage 1 — Directory Discovery
Scans the base URL against the full wordlist. Collects everything that
isn't 404, 500, or 0.

### Stage 2 — Subdirectory Expansion
Takes all 200s from stage 1 and scans each one against the full wordlist.
Runs all directory expansions in parallel — one goroutine per directory.

### `Run` is now a clean orchestrator
```go
func Run(req model.Request) error {
    if err := validate(&req); err != nil {
        return err
    }
    base := strings.TrimRight(req.URL, "/")
    client := newClient()
    bf, err := buildBaseline(client, base)
    if err != nil {
        return err
    }
    bf.print()

    fmt.Printf("═══ Stage 1 — Directory Discovery ═══\n\n")
    r1 := stageOne(client, base, req.FilePath, bf)

    r2 := newScanResult()
    if len(r1.dirs) > 0 {
        fmt.Printf("\n═══ Stage 2 — Subdirectory Expansion (%d directories) ═══\n\n", len(r1.dirs))
        r2 = stageTwo(client, r1.dirs, req.FilePath, bf)
    }

    all := mergeResults(r1, r2)
    if len(all) > 0 {
        saveResults(req.URL, all)
    }
    return nil
}
```

## Dual Result Maps

`scanResult` carries two maps populated in the same worker pass — no
second lookup needed:

```go
type scanResult struct {
    hits map[string]int  // url -> status — everything we like (not 404/500/0)
    dirs map[string]bool // url -> true — only 200s, for stage 2 seeding
}
```

Both populated atomically in the worker:
```go
if res.status != 404 && res.status != 500 && res.status != 0 {
    result.hits[target] = res.status
    if res.status == 200 {
        result.dirs[target] = true
    }
}
```

Stage 2 iterates `r1.dirs` directly — no filtering, no map lookups.

## Baseline Type

Setup noise extracted from `Run` into dedicated types and helpers:

```go
type baseline struct {
    soft404     bool
    trulyStatic bool
    b1          probeResult
}

func buildBaseline(client *http.Client, base string) (baseline, error)
func (b baseline) print()
```

`scanBase`, `stageOne`, and `stageTwo` all accept `baseline` instead
of three separate `soft404`, `trulyStatic`, `baseline1` params.

## Hit Saving

### What gets saved
All non-404, non-500 responses — 200, 301, 302, 303, 401, 403.
A denied path is still a confirmed existing path.

### Save location
Written to the current working directory:
```
site-com_scan.txt
```

No subdirectory — file drops where cwrap was invoked, immediately
usable with `cwrap recon --tfile site-com_scan.txt`.

### Format
Bare URLs, one per line — no status codes, no byte counts:
```
http://site.com/admin
http://site.com/dashboard
http://site.com/api/admin/users
```

## Default Wordlist

When no wordlist is provided, cwrap resolves one relative to the binary:

```go
func defaultWordlist() string {
    exe, _ := os.Executable()
    exe, _ = filepath.EvalSymlinks(exe)
    return filepath.Join(filepath.Dir(exe), "wordlists", "directory-list-2.3-medium.txt")
}
```

`filepath.EvalSymlinks` handles the temp binary path created by `go run`.
