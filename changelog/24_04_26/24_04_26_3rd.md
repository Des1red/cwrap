## New Command: `scan`

```bash
cwrap scan <url> <wordlist>
cwrap scan <url>               # uses default wordlist next to binary
```

### Architecture

```
internal/scan/
├── scan.go     — Run, baseline probing, worker pool, hit collection
└── print.go    — formatResult, saveResults
```

### Intent layer

**`intent/scan.go`** — `ScanHandler`:
- Translates cookie and bearer tokens from word tokens
- No wordlist handling — path comes from `req.FilePath` set by `Parse`

**`intent/parse.go`** — `Parse` updated:
- `scan` case extracts URL from `args[2]` and wordlist path from `args[3]`
- Both stored on `model.Request` — `URL` and `FilePath`

**`runner/scan.go`** — `ScanExecutor`:
```go
type ScanExecutor struct{}
func (ScanExecutor) Run(req model.Request) error {
    return scan.Run(req)
}
```

## Soft 404 Detection

The most common problem with directory scanners is false positives —
servers that return 200 for every path including non-existent ones.

### Two-baseline strategy
cwrap probes two distinct random paths before scanning:

```go
baseline1, _ := probe(client, base+"/cwrap-xqzjk-404check")
baseline2, _ := probe(client, base+"/cwrap-zvmpt-404check")
```

The two baselines are compared:

| Condition | Strategy |
|-----------|---------|
| Both 404 | No soft 404 — show all non-404 results |
| Both 200, same hash | Static soft 404 — filter by exact content hash |
| Both 200, different hash | Dynamic soft 404 — filter by size band (±5%) |

### Static soft 404 (most common)
Server returns the same error page for everything. Baseline hashes match.
Results whose hash equals the baseline hash are silently skipped.

### Dynamic soft 404
Server randomizes responses (adds timestamps, tokens, request IDs).
Baseline hashes differ. Results within 5% of baseline size are skipped.

### Output
```

Scanning http://site.com

[200] http://site.com/admin (107 bytes)
[200] http://site.com/dashboard (103 bytes)
```

## Parallel Execution

20 goroutines share a single `http.Client` with a pooled transport:

```go
const workers = 20

Transport: &http.Transport{
    MaxIdleConns:        workers,
    MaxIdleConnsPerHost: workers,
}
```

A mutex serializes `fmt.Println` and `hits` appends — output doesn't
interleave across workers. Result order is non-deterministic (arrival order).

## Result Saving

Only HTTP 200 URLs are saved — bare URLs, one per line:

```
http://site.com/admin
http://site.com/dashboard
```

Filename derived from target URL:
```go
strings.NewReplacer("https://", "", "http://", "", "/", "_", ":", "-").Replace(rawURL)
```

Output: `site-com_scan.txt` — directly consumable by `cwrap recon --tfile`.

## Default Wordlist

When no wordlist is provided, cwrap looks for a default next to the binary:

```go
func defaultWordlist() string {
    exe, _ := os.Executable()
    exe, _ = filepath.EvalSymlinks(exe)
    return filepath.Join(filepath.Dir(exe), "wordlists", "directory-list-2.3-medium.txt")
}
```

Uses `filepath.EvalSymlinks` to resolve temp paths created by `go run`.
Falls back to an error if the file doesn't exist.

|-------|---------|

| Red | 401/403 — access denied |
| (none) | Everything else — skipped |

## scan → recon Pipeline

```bash
# Step 1 — discover paths
cwrap scan https://site.com wordlist.txt

# Step 2 — deep recon each discovered path
cwrap recon --tfile site-com_scan.txt http
```
