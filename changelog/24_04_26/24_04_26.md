## Credentialless Expander

### Logic
1. Find all stage 1 confirmed credentialless steps
2. Rebuild the credentialless token jar from the source entity
3. Target entities with `RoleBoundary` or `AdminSurface` signal not
   already tested in stage 1
4. Filter out entities with only 404 responses during recon —
   not meaningful targets

### Filter: `hasSuccessfulReconResponse`
Excludes entities whose entire recon status history is 404. Requires at
least one non-404 response to qualify as a stage 2 target.

```go
func hasSuccessfulReconResponse(ent report.ReportEntity) bool {
    for code := range ent.Statuses {
        if code != 404 {
            return true
        }
    }
    return false
}
```

### New: `Statuses map[int]int` on `ReportEntity`
Report loader now parses `Statuses:` line into a map of status code to
count. Used by the credentialless filter.

## IDOR Expander

### Logic
1. Seed from confirmed `KindIDOR` steps AND confirmed `KindOwnership` steps
2. For ownership seeds, find the denied identity (not the owner) and use
   their token for enumeration
3. Enumerate neighboring object IDs (`/api/notes/1` → `/api/notes/2..21`)
4. Skip own objects via `isOwnObject`
5. Stop at 3 consecutive 404 or 403 responses

### Own-object detection: `isOwnObject`
Parses JSON response body properly — no string matching:

```go
var ownershipFields = []string{
    "owner_id", "OwnerID",
    "user_id", "UserID",
    "author_id", "AuthorID",
    "created_by", "CreatedBy",
    "owned_by",  "OwnedBy",
}
```

- Handles numeric and string uid values
- Handles both object and array responses (checks first element)
- Handles camelCase and snake_case field names
- No false positives from partial string matching

### Denied identity resolution for ownership seeds
```go
func findDeniedIdentityForOwnershipSeed(session, seed) string
```
Looks up the ownership param from the target entity, finds an identity in
`IdentityDenied` that differs from the confirmed identity. The denied
identity is the one to enumerate with — they were blocked during recon but
may have access to neighboring objects.

### Token claim resolution
After switching to denied identity, resolves actual JWT claims from vault
to ensure role/uid reflects the correct token principal:

```go
if jar := tokens.FindIdentityCookies(session.Recon, deniedID); jar != nil {
    if r, u := tokens.ParseJWTPrincipalUnsafe(tokens.FirstCookieValue(jar));
        r != "" || u != "" {
        modified.Role = r
        modified.UserID = u
    }
}
```

## Ownership Expander

### Logic
1. Seed from confirmed `KindOwnership` steps
2. Use the confirmed (bypassing) identity's token
3. Enumerate neighboring object IDs from the confirmed URL
4. Skip own objects via `isOwnObject`
5. Stop at 3 consecutive 404 or 403 responses

## Enumeration Guard

### Fix: Consecutive miss counter includes 403
Previously only 404 triggered the miss counter. Servers that return 403
for non-existent objects beyond the valid range would cause runaway
enumeration. Now both 404 and 403 increment the counter:

```go
if res.StatusCode == 404 || res.StatusCode == 403 {
    consecutiveMisses++
} else {
    consecutiveMisses = 0
}
if consecutiveMisses >= 3 {
    break
}
```
