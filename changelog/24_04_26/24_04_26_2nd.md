## Signal Attribution Fixes

### Root cause
Three signal tagging bugs caused false positives on wrong entities.
All were in HTML/JS extraction — signals were being tagged on the
**source** entity (the page containing the link/form) instead of the
**target** entity (the endpoint the signal belongs to).

### Fix 1: `extractHTML` — `SigAdminSurface` on link discovery
```go
// before — tagged the page that contains the link
if isSensitivePath(link) {
    ent.Tag(knowledge.SigAdminSurface)
}

// after — tags the target endpoint
if isSensitivePath(link) {
    e.k.Entity(link).Tag(knowledge.SigAdminSurface)
}
```

### Fix 2: `extractHTML` — form method and state attribution
`ent.AddMethod(method)` and `ent.Tag(SigStateChanging)` were adding
the form's HTTP method to the page containing the form instead of the
action target. The method and state-change belong to the action endpoint.

```go
// before — attributed to source page
ent.AddMethod(method)
if method == "POST" || fi.IsDestructive {
    ent.Tag(knowledge.SigStateChanging)
}

// after — attributed to action target
target := e.k.Entity(link)
target.AddMethod(method)
if method == "POST" || fi.IsDestructive {
    target.Tag(knowledge.SigStateChanging)
}
```

### Fix 3: `handleJSEndpoints` — `SigAdminSurface`, `SigStateChanging`, `AddMethod`
Same issue — all three were on the JS source entity instead of the
discovered endpoint:

```go
// before
ent.Tag(knowledge.SigAdminSurface)
ent.Tag(knowledge.SigStateChanging)
ent.AddMethod(ep.Method)

// after
e.k.Entity(link).Tag(knowledge.SigAdminSurface)
e.k.Entity(link).Tag(knowledge.SigStateChanging)
// AddMethod removed entirely — let probe result confirm it
```

`AddMethod` was also removed from JS endpoint discovery because pre-adding
a method based on JS analysis skips verification in `expandMethods`. The
probe result now confirms the method when the response returns an allowed
status code.

### Impact
Before fix:
```
AdminSurface: 2   ← /home and / falsely tagged
```
After fix:
```
AdminSurface: 1   ← only /login/admin correctly tagged
```

## New: `SigPublicAccess` Signal

### Detection
Added `detectPublicAccess` called in `runQueuedProbes` alongside other
detectors. Tags an entity as public when the `anonymous` identity gets
200 without sending any credentials:

```go
func (e *Engine) detectPublicAccess(ent *knowledge.Entity, identityStatuses map[string]int) {
    id := ent.Identities[knowledge.Anonymous]
    if id == nil {
        return
    }
    if id.Kind == knowledge.IdentityNone && identityStatuses[knowledge.Anonymous] == 200 {
        ent.Tag(knowledge.SigPublicAccess)
    }
}
```

### Report output
Global stats section lists public endpoint count and URLs:
```
Public endpoints:  1
  - http://localhost:9001
```

### Summary output
```
Public endpoints:    1
```

### High risk highlight
```go
if ent.SeenSignal(knowledge.SigPublicAccess) && ent.SeenSignal(knowledge.SigAdminSurface) {
    out = append(out, fmt.Sprintf("Public admin surface: %s", ent.URL))
}
```

### Chain filter
Stage 2 credentialless expansion skips public endpoints — confirming
access on a public endpoint is not a finding:
```go
if hasSignal(ent, knowledge.SigPublicAccess) {
    continue
}
```

### Summary bug fix
`fmt.Printf` replaced with `fmt.Fprintf(w, ...)` to correctly write
to the provided writer instead of stdout. Public count loop now iterates
`urls` (sorted) instead of `k.Entities` (unordered map).

## Chain: All Identities Coverage

### Problem
Three places only tested the first matching identity:
- Credentialless expander used only the first credless identity
- IDOR expander used only the first denied identity from ownership seeds
- Ownership expander had the 403 consecutive miss guard missing

### Fix 1: Credentialless expander — all credless identities
`FindAllCredlessIdentities` returns all identities with issued tokens
but no credentials. Each is expanded independently with its own
stage1 history check:

```go
credlessIDs := find.FindAllCredlessIdentities(*sourceEnt)
for _, credlessID := range credlessIDs {
    // per-identity targets, per-identity alreadyTested map
    targets := stage2CredentiallessTargets(session.Recon, session, credlessID.Name)
    // ...
}
```

Per-identity role/uid also resolved from actual token claims rather
than copied from seed.

### Fix 2: IDOR expander — all denied identities from ownership seeds
`findAllDeniedIdentitiesForOwnershipSeed` returns all denied identities
for the param, not just the first:

```go
func findAllDeniedIdentitiesForOwnershipSeed(session, seed) []string
```

Each denied identity is now seeded independently for enumeration.

### Fix 3: Ownership expander — 403 miss guard
Added 403 to the consecutive miss counter alongside 404:
```go
if res.StatusCode == 404 || res.StatusCode == 403 {
    consecutiveMisses++
    if consecutiveMisses >= 3 {
        break
    }
}
```

Without this, servers that return 403 for non-existent objects beyond
the valid range caused runaway enumeration (21+ requests observed).

## isOwnObject Rewrite

### Problem
Previous string matching approach (`strings.Contains(s, '"owner_id":1')`)
was fragile — missed camelCase fields, float64 JSON numbers, and array
responses.

### New implementation
Proper JSON parsing with ownership field priority list:

```go
var ownershipFields = []string{
    "owner_id", "OwnerID",
    "user_id", "UserID",
    "author_id", "AuthorID",
    "created_by", "CreatedBy",
    "owned_by", "OwnedBy",
}
```

- Handles `float64`, `string`, `json.Number` uid values
- Handles object and array responses (checks first element)
- No false positives from partial string matching

## Chain: `hasSuccessfulReconResponse` Filter

Credentialless stage 2 targets now exclude entities whose entire recon
status history is 404 — these are path variants that don't exist as
real endpoints:

```go
func hasSuccessfulReconResponse(ent report.ReportEntity) bool {
    for code := range ent.Statuses {
        if code != 404 {
            return true
        }
    }
    return false
}
```

`Statuses map[int]int` added to `ReportEntity` and parsed from the
`Statuses:` line in the report loader.
