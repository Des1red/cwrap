# cwrap recon — param attribution & signal accuracy

`21/04/26` `v2.3.4`

## Overview

Fixed incorrect parameter attribution and improved signal accuracy by
separating execution context from ownership context. Path parameters are now
analyzed on their logical parent entity instead of individual path instances.

---

## Fix — Correct parameter ownership attribution

### Problem

Path parameters (e.g. `/notes/{id}`) were recorded on the target entity
(`/notes/1`) instead of the source entity (`/notes`).

This caused:

* Split parameter intelligence across entities
* Broken ownership detection
* Weak IDOR correlation
* Incorrect interest scoring

### Change

```go
p := accumEnt.Params[k]
```

(replacing)

```go
p := ent.Params[k]
```

### Impact

* Parameter intelligence aggregates correctly
* Ownership boundaries become reliable
* IDOR detection improves
* Cleaner reports (no duplicated params)

---

## Fix — Proper accumulation for path parameters

### Change

```go
accumEnt := ent
if isPathParam && probe.SourceURL != "" {
    accumEnt = kg.Entity(probe.SourceURL)
}
```

### Impact

* Path params are attributed to their parent endpoint
* Cross-value analysis (IDOR / ownership) becomes consistent

---

## Fix — Exclude session identity from stats

### Problem

`session` identity polluted access/denied counters, affecting ownership and
IDOR signals.

### Change

```go
func shouldReportIdentity(name string) bool {
    return name != "session"
}
```

Applied to identity counters.

### Impact

* Only real identities influence analysis
* Cleaner access/denied maps
* More accurate ownership detection

---

## Feature — Probe impact learning

### Problem

No direct signal that a parameter influenced responses during probing.

### Change

Added lightweight learning:

```go
if fp != ref {
    p.Interest++
    p.ObservedChanges["input-affects-response"] = true
}
```

### Impact

* Parameters that change behavior gain interest
* Better prioritization for fuzzing and analysis
