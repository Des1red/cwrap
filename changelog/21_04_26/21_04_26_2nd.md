# cwrap — test suite
`21/04/26` `v2.3.3`

## Overview
First test suite for cwrap. Covers the core signal pipeline, both recon
engines, all pure helper functions, and the CLI layer. 11 packages with
tests, all passing. Tests use standard `testing` only — no external
dependencies.

## `cwrap/testutil`

Shared helpers used by engine tests.

- `MockResp(status, contentType, body)` — builds a minimal `*http.Response`
  with a readable body, suitable for passing to `learn` and other functions
  that consume HTTP responses without making network calls
- `MockRespWithHeader(status, contentType, headerKey, headerVal, body)` —
  same as `MockResp` with an additional header set, used for redirect tests
- JSON fixtures: `UserJSON`, `UsersJSON`, `PostJSON`, `ErrorJSON`,
  `ForbiddenJSON`
- HTML fixtures: `LoginHTML`, `HomeHTML`

## `canonicalize` — 14 tests

Tests for all three normalization functions.

**`JSON`**
- Same schema different values normalizes equal
- Different schemas (extra key) normalizes different
- Different value types (bool vs string) normalizes different
- Arrays collapsed to first element — list of 3 vs list of 1 equal
- Empty arrays equal
- Nested objects with same schema equal
- Invalid JSON returns error
- Null values preserved

**`HTML`**
- Same DOM structure different text content normalizes equal
- Different DOM structures normalize different
- Script content stripped — different inline tokens normalize equal
- Attribute values stripped — different hrefs normalize equal

**`StripNumbers`**
- Digits replaced with `#`
- No digits — unchanged
- Does not mutate input slice

## `paramintel` — 14 tests

Tests for `ClassifyParam`.

- `id` exact match → `IDLike`
- `_id` suffix → `IDLike`
- `user`, `account` substring → `IDLike`
- Common non-ID names (`title`, `body`, `email`) → not `IDLike`
- `_token`, `session_`, `auth_`, `_key` → `TokenLike`
- Non-token names → not `TokenLike`
- `debug`, `test_`, `dev_`, `preview` → `DebugLike`
- Non-debug names → not `DebugLike`
- `IDLike` sets `SigIDLikeParam` on entity
- `TokenLike` sets `SigTokenLike` on entity
- `DebugLike` sets `SigDebugFlag` on entity
- Multi-classification param (`auth_user_id`) gets both `IDLike` and `TokenLike`
- Unclassified param (`title`) has no flags

## `jsintel` — 24 tests

Tests for `Learn`.

**Endpoint discovery**
- `fetch("/path")` → GET endpoint
- `fetch('/path')` single quotes
- `axios.get`, `axios.post`, `axios.delete` → correct method
- `xhr.open("POST", "/path")` → POST endpoint
- `xhr.open("GET", "/path")` → GET endpoint
- Path literal `/api/...` → discovered
- Non-interesting path `/static/...` → filtered out
- Duplicate endpoints across `fetch` + `axios` → deduplicated to one

**Secret detection**
- JWT token → `SigSensitiveKeyword`, `jwt` finding, leak recorded
- AWS access key `AKIA...` → `SigSensitiveKeyword`, `aws_key` leak
- PEM private key block → `SigSensitiveKeyword`, `pem` leak
- `api_key = "..."` assignment → `keyword` leak
- `password = "..."` assignment → `keyword` leak

**Privilege surfaces**
- `user.role === "admin"` → `role_check` finding
- `isAdmin === true` → `priv_flag` finding
- `if (isAdmin)` gate → `priv_gate` finding

**Environment variables**
- `process.env.API_URL` → `env_ref` finding
- `import.meta.env.VITE_API_KEY` → `env_ref` finding
- `NEXT_PUBLIC_*` → `env_public` finding

**Internal infra**
- `.internal` domain → `host_internal` finding
- RFC1918 IP `192.168.x.x` → `host_private_ip` finding
- `https://api.example.com/...` → `host_url` finding

**Feature flags**
- `FEATURE_*` token → `feature_token` finding
- `flags = { ... }` object → `feature_block` finding

**Edge cases**
- Nil entity → nil return, no panic
- Empty body → no endpoints, no leaks
- Duplicate leaks deduplicated by kind+key+value

## `behavior` — 24 tests

**Pure helpers**
- `looksLikePathID`: integers, UUIDs, strings, empty, too long
- `pathTemplate`: numeric segment, UUID segment, no IDs, multiple IDs,
  root path
- `discoveryValuesFor`: IDLike, page, limit, offset, is_ prefix, enabled,
  sort, order_by, q, search, generic unknown
- `methodAllowed`: all allowed codes, 405/501 blocked, unknown 999 blocked
- `extractCurrentValue`: present key, multiple params, absent key, no query
- `isElevatedRole`: admin/owner/root/super/staff/mod → true,
  member/user/viewer/guest → false
- `parseJWT`: valid token extracts claims, malformed formats return nil

**Detectors**
- `detectAuthBoundary`: authed 200 + unauthed 401 → `SigAuthBoundary`,
  authed 200 + unauthed 403 → `SigAuthBoundary`,
  no authed success → no signal
- `detectRoleBoundary`: authed 403 → `SigRoleBoundary`,
  unauthed 403 only → no signal

**Analyzers**
- `analyzeOwnership`: mixed access → `OwnershipBoundary` + `SigObjectOwnership`,
  both succeed → no boundary,
  only one authed identity → no boundary (need ≥2)

**Probe impact**
- `learnProbeImpact`: changed fingerprint → `Interest++` + `input-affects-response`,
  same fingerprint → no change,
  injected-only param → skipped
