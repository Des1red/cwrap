# cwrap recon ‚Äî report improvements
```19/04/26```
## Overview

Report rewrite focused on signal-to-noise ratio. The full report previously
printed every field regardless of value ‚Äî false booleans, empty maps, zero
integers, injected params with no evidence. Now only meaningful non-default
data is printed.

---

## Fix 1 ‚Äî Full report rewrite

### Problem
The full report was a raw struct dump. A single entity with no interesting
findings produced 80+ lines of `false`, `(none)`, and `Interest: 0`. The
actual findings were buried at the bottom after scrolling through noise.

### Changes

**`report/full.go`** ‚Äî complete rewrite of `writeEntityDetails`:

**Entity header** ‚Äî condensed to essential fields only:
- `Probes: N` replaces the full State block
- `Methods: [...]` as a single line instead of a list
- `AuthLikely` and `CSRFPresent` only printed when `true`
- Content type printed as a single word (`HTML`, `JSON`, `XML`) not three booleans
- Statuses as compact inline format: `200√ó43 303√ó51` instead of a block

**Session** ‚Äî entire section skipped if nothing happened (`SessionUsed`,
`SessionIssued`, and `SessionCookies` all empty/false)

**Identities** ‚Äî each identity is now one line with tags:
`baseline: [IdentityUser creds role=member effective]`
Tags only include non-default values ‚Äî `creds`, `rejected`, `issued-token`,
`effective`, `role=X`, `scheme=X`, `csrf`. Cookie names printed on a
second line only when present.

**Parameters** ‚Äî injected params with zero evidence are skipped entirely.
A param has evidence if it has: any true heuristic flag, any observed change,
any identity denial, mixed access+denial, or non-zero interest.
Params with evidence print as a single compact line:
`id: [injected IDLike interest=2]`
With sub-lines for `changes`, `access`, `denied` only when non-empty.

**JS Intelligence** ‚Äî section skipped entirely when empty

**Findings** ‚Äî prefixed with `!` for visual scanning

**Next Steps** ‚Äî prefixed with `>` for visual scanning

Added helper functions:
- `paramHasEvidence(p *ParamIntel) bool`
- `hasNonInjectedParams(ent *Entity) bool`
- `paramSourceShort(p *ParamIntel) string`

## Fix 2 ‚Äî Unauthenticated access false positive in findings

### Problem
`deriveFindings` fired "Unauthenticated access observed" for any injected
param where `anonymous` got a 200. On public pages like `/login`, every
injected param got a 200 from everyone ‚Äî the finding was meaningless noise.

### Changes

**`report/derive.go`**
- Unauthenticated access finding now requires `len(p.IdentityDenied) > 0` in
  addition to `p.IdentityAccess["anonymous"] > 0`
- Access without denial on an injected param is not a finding

---

## Fix 3 ‚Äî `RoleBoundary` detection with expired tokens

### Problem
`detectRoleBoundary` only checked for status 403. When the session token was
expired, the server returned 401 instead of 403 to the baseline identity ‚Äî
the signal never fired even though a credentialed identity was clearly denied.

### Changes

**`behavior/run.go`** ‚Äî `detectRoleBoundary`:
- Now checks `status != 401 && status != 403` instead of `status != 403`
- Both 401 and 403 from a credentialed identity are evidence of a role/auth
  boundary ‚Äî an expired token that was legitimately obtained still means
  "sent credentials, got denied"
