## Identity System

### Rename: `baseline` ŌåÆ `session`
The live session identity is now named `session` to accurately reflect what
it is ŌĆö the current floating session that rotates as probing progresses.

### Dynamic identity naming
Discovered identities are now named by role and uid:
```
member-uid-1, member-uid-2, owner-uid-3
```
Previously used `role-member`, `role-owner` ŌĆö which would collide when two
members exist and carried no uid disambiguation.

### Fix: Live identity cookie snapshots frozen
`captureSession` previously overwrote live identity cookies on every probe.
Bob's cookies would be replaced with Alice's token when any identity probed
`/login/alice`. Fixed by returning early for any non-`session` identity:

```go
if idMeta.Name != "session" {
    e.discoverIdentityFromResponse(resp)
    return
}
```

### Extract: `discoverIdentityFromResponse`
JWT claim parsing and new identity registration extracted from `captureSession`
into a standalone helper. Both `session` and live identity code paths call it,
ensuring new accounts are discovered regardless of which identity is probing.

### Fix: 403 no longer marks identity as Rejected
`extractIdentity` previously marked any 403 response as `id.Rejected = true`.
403 means valid identity, wrong permission ŌĆö not invalid credentials. Only 401
now sets `Rejected`. This fixed Bob showing as `IdentityInvalid` on endpoints
he doesn't own.

## Parameter Attribution

### Fix: Path params accumulate on source entity
`storeResponse` now routes path param accumulation to the source entity
(the one that ran `expandPathIDs`) rather than the target variant entity.

```go
accumEnt := ent
if isPathParam && probe.SourceURL != "" {
    accumEnt = kg.Entity(probe.SourceURL)
}
```

### Fix: `accumEnt.Params[k]` instead of `ent.Params[k]`
Access/denied maps now attribute to the correct entity ŌĆö the one that owns
the param, not the path variant that happened to execute the probe.

### Fix: `session` excluded from access/denied maps
`shouldReportIdentity` filters `session` from `IdentityAccess` and
`IdentityDenied` counters. The floating session identity would otherwise
pollute ownership and IDOR analysis.

```go
func shouldReportIdentity(name string) bool {
    return name != "session"
}
```

## IDOR Detection

### Fix: `analyzeIDOR` on accumulated target data
Added a second `analyzeIDOR` call outside the path ID probe guard, using
accumulated data on the target entity:

```go
e.analyzeIDOR(target, target.AccumResponses, target.AccumStatuses)
```

This enables IDOR detection for query param endpoints (`?user_id=N`) where
multiple probe values accumulate on the same target entity.

### Fix: Path ID probes rerun after new identity discovery
`addLiveIdentity` now clears path template dedup state and re-queues path ID
expansion for all seen entities with path params when a new identity is found.

Changes in `addLiveIdentity`:
1. Clear `probedPathTemplates` map entirely
2. For each entity with path params: reset `PathIDProbed = false`
3. Call `clearSeenPathIDProbeFamily` to clear dedup keys from both
   `root.SeenProbes` and `ent.SeenProbes`
4. Re-queue entity for expansion at priority 155

### Fix: `clearSeenPathIDProbeFamily` uses path template matching
Replaces prefix-based matching with `pathTemplate()` normalization:

```
/api/users/1  ŌåÆ  /api/users/{id}
/api/users/2  ŌåÆ  /api/users/{id}  (same family)
/api/users/profile  ŌåÆ  /api/users/profile  (different, not cleared)
```

Prevents cross-endpoint probe key collisions when routes share a prefix.

### Fix: `isComparableIdentity` excludes `session`
Ownership and IDOR analysis now only compares real discovered identities.
`session` is excluded because it rotates cookies and doesn't represent a
stable user perspective.

```go
func isComparableIdentity(name string, id *knowledge.Identity) bool {
    if id == nil || !id.SentCreds { return false }
    if name == knowledge.LiveSession { return false }
    return true
}
```

## JS Intelligence

### Fix: `JSAnalyzed` guard on external JS files
`jsintel.Learn` now runs at most once per JS entity regardless of how many
times the file is fetched across identities and probe cycles:

```go
if looksLikeJS(url, resp) && !ent.State.JSAnalyzed {
    ent.State.JSAnalyzed = true
    jsEndpoints := jsintel.Learn(ent, url, body)
    e.handleJSEndpoints(ent, url, jsEndpoints)
}
```

### Fix: `JSAnalyzed` guard on inline scripts
Same guard applied to inline `<script>` blocks in `extractHTML`. HTML entities
were accumulating inflated endpoint counts because every probe cycle re-ran
inline script analysis.

Before: `endpoint: 36` on every HTML entity  
After: `endpoint: 1` (the actual single inline fetch call)

### Added: `JSAnalyzed bool` to `EntityState`
```go
type EntityState struct {
    // ...existing fields...
    JSAnalyzed bool
}
```

## Report

### Fix: False positive "Unauthenticated access observed"
`buildHighRiskHighlights` previously flagged any entity where anonymous got
200. On open APIs this fired on every endpoint. Now requires mixed access ŌĆö
anonymous succeeds AND at least one identity is denied:

```go
if p.IdentityAccess["anonymous"] > 0 && len(p.IdentityDenied) > 0 {
    // flag as finding
}
```
