# cwrap — session expiry filtering & help menu
`20/04/26` `v2.2.3`

## Overview

Two small quality-of-life fixes: expired JWT cookies are now filtered at
session load time so stale tokens never reach the engine, and the help menu
was reorganized to be clearer about what each command and word does.

---

## Fix 1 — JWT expiry filtering at session load

### Problem
`session.Load` returned all stored cookies regardless of expiry. Expired JWT
tokens from previous runs (or from a different app on the same host) were
injected into the baseline identity and sent to the target. The server rejected
them with 401, triggering the stale session recovery path unnecessarily. Worse,
on targets where the root returns 200 to everyone (SPAs), the stale session
detection never fired and expired tokens were silently carried through all
probes.

### Changes

**`session/store.go`**
- Added `isExpiredJWT(val string) bool` — parses the payload of any
  JWT-shaped value (`x.y.z`) and checks the `exp` claim against
  `time.Now().Unix()`
- Non-JWT values (opaque tokens, csrf tokens) return `false` — pass through
- In `Load`, after unmarshalling the store, iterate all identity cookies and
  delete any where `isExpiredJWT` returns `true`
- Expired cookies are removed before the store is returned — the engine never
  sees them
- Added imports: `encoding/base64`, `encoding/json`, `strings`, `time`

## Fix 2 — Help menu reorganization

### Problem
The help menu was a single flat list of "semantic words" mixing encoding,
profiles, data, auth, and behavior words together. Recon had minimal
documentation — one example line and a small profile table.

### Changes

**`logger/help.go`**
- Sections now: Commands, Recon profiles, Recon flags, Encoding words,
  Profile words, Data words, Authentication words, Behavior words
- Each section is focused — no more mixed flat list
- Recon gets dedicated profile and flag sections
- Examples split by command — fetch, send, upload, recon each have their own
  block with multiple cases
- Nested JSON and array syntax shown explicitly in Data words
- `auto-cookie` and `csrf` restored to Authentication words section
- Escape hatch section cleaned up with real example values
