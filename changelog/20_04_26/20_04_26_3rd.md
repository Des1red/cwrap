# cwrap recon — dynamic identity discovery
`20/04/26` `v2.3.0`

## Overview

cwrap can now discover new probe identities mid-run by detecting JWT tokens
with role/uid combinations that differ from the baseline. When a new identity
is found, it is immediately added to the active probe set and all known
entities are re-queued for probing with it. This unlocks ownership boundary
detection and cross-role IDOR analysis in a single run without manual
credential setup.

---

## Feature — Dynamic live identity discovery

### Problem
The engine had three fixed synthetic identities (baseline, anonymous,
fake-admin) and one real identity (baseline). Without a second real
authenticated identity, `analyzeOwnership` was permanently dead and IDOR
detection was limited to structural response diffs. There was no way to
compare behavior between two real users with different roles.

### Design
During probing, whenever a server issues JWT cookies, the engine parses the
claims and checks if the `role|uid` combination is genuinely new. If it is,
a new real probe identity is created from those cookies and added to the live
identity set mid-run. All previously seen entities are immediately re-queued
for probing with the new identity.

### Changes

**`behavior/engine.go`**
- Added `knownRoleUIDs map[string]bool` — tracks all role|uid combos that
  are already represented by a real identity
- Added `discoveredIdentities map[string]bool` — prevents the same role|uid
  from spawning multiple identity objects
- Both initialized in `New()`

**`behavior/run.go`**
- After baseline identity extraction, registers `baseline.Role|baseline.UserID`
  into `knownRoleUIDs` before `captureSession` runs — prevents the baseline
  role from being re-added as a live identity

**`behavior/session.go`** — `captureSession`:
- After saving cookies, parses any JWT-shaped cookies from the response
- Extracts `role` and `user_id` claims
- If `role|uid` is not in `knownRoleUIDs` and not in `discoveredIdentities`
  — calls `addLiveIdentity`

**`behavior/identity.go`** — new `addLiveIdentity(name, cookies, roleUID)`:
- Guards against duplicate identity names
- Creates a cookie snapshot (stable closure)
- Registers `roleUID` into `knownRoleUIDs`
- Appends a new `Identity` with `Apply` that injects the snapshotted cookies
- Re-queues all currently seen entities as GET identity probes (priority 160)
- Debug output: `== New identity discovered: role-X ==`

**`internal/recon/http/html.go`** — link probing:
- `<a href="">` links now push a GET probe via `e.k.PushProbe` in addition to
  adding a graph edge
- Previously links were only recorded as edges — they were never actually
  fetched

**`internal/recon/knowledge/constants.go`**
- Added `ReasonLinkProbe = "link"`

## Result on test app

Single run starting from `/login` (no session file):

1. First probe to `/login/submit` issues `member|1` token
   → `role-member` identity created
2. Engine re-queues all seen entities with `role-member`
3. First probe to `/login/admin` issues `owner|42` token
   → `role-owner` identity created
4. Engine re-queues all seen entities with `role-owner`
5. All subsequent probes run with 5 identities:
   `baseline, anonymous, fake-admin, role-member, role-owner`

Report shows `/admin` probed by all 5 identities with correct role/uid/expiry
in a single run. `analyzeOwnership` can now fire when roles differ on
object-level endpoints.
