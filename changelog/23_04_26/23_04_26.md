## Report Loader

### Fix: Section-aware parser
`LoadReport` now tracks current section (`identities`, `parameters`,
`session`, `""`) to correctly distinguish identity lines from param lines.
Previously both matched `": ["` causing identity lines to be parsed as
params and access/denied maps to be lost.

### Fix: `changes:` line guard
Added explicit skip for `changes: [...]` lines in the parameters section —
these were matching the param line pattern and overwriting `currentParam`.

### New: Parse helpers
`LoadReport` split into focused helpers:

- `parseSignals` — parses `Signals: [...]` line
- `parseSessionCookie` — parses `    - name=value` line
- `parseIdentityLine` — parses identity tag line, extracts role/uid/CSRF
- `parseParamLine` — parses param tag line, sets boolean flags
- `parseCookiesLine` — parses `cookies: [...]` line under identity

## Probe: Credentialless

### Implemented: `CredentiallessProbe.Run`
1. Finds first identity with `issued-token` but no `creds` tag
2. Builds cookie map from entity session block
3. Finds all `AuthBoundary` entities in report
4. Replays token against each protected endpoint
5. Prints `[CONFIRMED]` with response body preview on 200
6. Prints `[blocked]` on 401/403

### New: Identity metadata in output
```
  identity: anonymous (role=member uid=1)
  token:    auth_token=eyJ...
```
Role and uid sourced from `find.FindIdentityMeta`.

### New: Response body preview on 200
```
  [CONFIRMED] http://localhost:9001/api/notes/1 → 200
  response:   {"id":1,"title":"Alice's private note",...}
```
Truncated to 200 characters.

## Probe: IDOR

### Implemented: `IDORProbe.Run`
1. Collects identities from `IdentityAccess` and `IdentityDenied` maps
2. Skips synthetic identities (`anonymous`, `fake-admin`) — no real sessions
3. For each denied credentialed identity:
   - Looks up cookies via `cookies.FindIdentityCookies`
   - Prints identity name with role/uid from `find.FindIdentityMeta`
   - Prints token being used
   - Makes GET request to entity URL with identity's cookies
   - Prints result with response body on 200

### Output example
```
  param:   note_id
  access:  [member-uid-2 owner-uid-3]
  denied:  [anonymous fake-admin member-uid-2]
  testing: member-uid-2 (role=member uid=2)
  token:   auth_token=eyJ...
  [CONFIRMED] member-uid-2 accessed http://localhost:9001/api/notes/1 → 200
  response:   {"id":1,"title":"Alice's private note","content":"Only Alice can see this","owner_id":1}

  ✓ IDOR confirmed — cross-identity object access succeeded
```

## New: `find` Package

### `FindCredlessIdentity`
Returns first identity with `issued-token` tag but no `creds` tag.

### `FindProtectedEntities`
Returns all report entities with `AuthBoundary` signal.

### `FindIdentityMeta`
Returns role and uid for a named identity by searching all report entities.
Used by both IDOR and credentialless probes to annotate output with
identity context.

---

## New: `cookies` Package

### `BuildCookieMap`
Matches identity cookie names against entity session cookies, returns
`map[string]string` of name→value pairs ready for injection.

### `FindIdentityCookies`
Searches all report entities for session cookies belonging to a named
identity. Returns first match found.
